**팀원들과 git 공유**

1. Organization 생성
2. Organization에 Repositories 생성.
3. Issues 생성
4. 프로젝트 연결
5. 브랜치 생성 및 코드 복사
6. 원하는 폴더에 터미널 생성
7. git clone "Repositories의 html 주소" 입력
8. 생성된 폴더에 터미널 생성
9. git fetch origin 입력
10. git checkout "브랜치명"
11. git branch로 연결 확인
12. npm create vite@latest "프로젝트명"으로 리엑트 생성
13. git add . // git commit -m "설명" // git push 로 데이터 업로드 (데이터 수정 후 업로드 하지않으면 에러 발생)
14. git checkout "브랜치명"으로 다른 브랜치로 이동 가능
15. git pull 로 업로드 된 파일 새로고침
